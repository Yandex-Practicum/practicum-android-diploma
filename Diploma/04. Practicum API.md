# API для поиска вакансий

Для финального задания мы подготовили специальный API — с его документацией можно ознакомиться этой ссылке. Там же вы найдёте всю информацию, необходимую для работы над приложением. Но, чтобы вы не упустили ничего важного, в этом уроке проведём небольшой обзор документации.

API предоставляет полный набор запросов, необходимых для реализации функциональных возможностей приложения:

- поиск среди вакансий (в том числе, с указанием дополнительных параметров для фильтрации);
- получение полной информации о вакансии по её идентификатору;
- получение списка географических областей (пригодится для реализации настроек фильтрации);
- получение списка сфер деятельности компании-работодателя — или просто индустрий (этот запрос тоже используется для настроек фильтрации).

Как отмечалось ранее, вакансии и сопутствующая им информация, которые вы будете получать от сервера, ненастоящие. Так, если изображение логотипа компании-работодателя ещё может соответствовать её реальному названию (Яндекс, Microsoft, Apple), то её адрес или сфера могут не иметь ничего общего с действительностью. То же самое касается и описания самих вакансий.

Так что этот API можно рассматривать как источник вакансий из параллельной вселенной, где компания Apple занимается производством в сфере тяжёлого машиностроения. Все совпадения с названиями компаний, существующих в нашем мире, случайны.

На сервере доступно более 200 тысяч вакансий. Этого более чем достаточно для проверки работы вашего приложения. Поиск производится по названию должности, которых предусмотрено всего десять:

- "Data Analyst",
- "Product Manager",
- "UX Designer",
- "Backend Developer",
- "Frontend Developer",
- "Full Stack Developer",
- "DevOps Engineer",
- "Cloud Architect",
- "Machine Learning Engineer",
- "Data Scientist",
- "Android Developer",
- "iOS Developer".

При выполнении поискового запроса сервер вернёт вакансии, в названиях должностей которых будет найдено совпадение с текстом, введённым в строке поиска. Учитывайте это при проверке корректности реализации задач.

Сам по себе API довольно простой и не содержит лишних запросов. Но есть пара нюансов, на которые стоит обратить внимание.

## Аутентификация

Наш API для поиска вакансий — закрытый. Это означает, что все выполняемые запросы должны быть авторизованными. Способ аутентификации вам уже знаком: здесь используется Bearer-токен.

Получить токен аутентификации очень просто:

1. Перейдите по ссылке из этого документа на страницу для получения токена.
2. В поле Username введите ваш адрес электронной почты.
3. Нажмите на кнопку Get Token.
4. Токен сгенерируется сразу — скопируйте и сохраните его для дальнейшего использования при авторизации выполняемых запросов.

## Формат заголовка

Чтобы выполнять условия авторизации запросов в API поиска вакансий, каждый запрос должен сопровождаться заголовком следующего вида:

```
Authorization: Bearer <token>
```

Вместо `<token>` вам необходимо подставить тот самый сгенерированный токен (без угловых скобок).

Поскольку сервер возвращает данные в JSON-формате, то следует добавить и вот такой заголовок:

```
Content-Type: application/json
```

Больше никаких дополнительных заголовков для работы с API не требуется.

Помимо информации о запросах и формате ответов, по ссылке на документацию вы можете найти небольшую инструкцию для работы с Postman.

Postman — это инструмент для тестирования API. Он представляет собой настраиваемый HTTP-клиент, через который можно выполнять запросы в любые API, получать ответы и анализировать их. Гибкость и возможности по настройке Postman позволяют работать с API любой сложности. Это очень удобный инструмент, который используется миллионами разработчиков и тестировщиков по всему миру.

Ознакомьтесь с инструкцией к Postman и попробуйте выполнить несколько запросов ещё до начала разработки, чтобы убедиться, что вы правильно работаете c API для поиска вакансий.

Далее, в секции с подсказками, вы найдёте ещё несколько полезных пояснений по работе с API.
